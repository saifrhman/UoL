\section{Definitions}
\subsection{Problem Modelling}

The task is a supervised multiclass classification problem predicting diagnosis (Depression, CI, or Both) from demographic and clinical features. Regression and clustering are unsuitable because the target variable is categorical and already labeled. The dataset contained 670 observations and 19 variables.

\subsection{Data Analysis and Feature Selection}

Initial exploration confirmed no missing values and no duplicate observations. Variables were classified by statistical meaning as per Figure~\ref{fig:variables}. This guided encoding and scaling decisions. The id column was removed as a unique identifier, and country was removed as a constant feature. A value of 99 in \textit{depression\_phq9\_score} lies outside the valid PHQ-9 clinical range 0 - 27 and was removed, leaving 669 observations. Class distribution analysis revealed moderate imbalance, with the Both class containing fewer observations, motivating the use of stratified sampling.

Figure~\ref{fig:variables} summarises the final variable classification used for preprocessing.

Visualisations explored the dataset. Histograms and violin plots examined numerical distributions, boxplots flagged potential outliers, and scatterplots revealed relationships with diagnosis. Fatigue tended to be higher in the CI and Both groups, while stress was distributed across all classes. Observations outside boxplot whiskers were inspected and retained as they remained within clinically valid ranges.

Class distribution analysis revealed moderate imbalance, with the \textit{Both} class containing fewer observations (Figure~\ref{fig:classdist}). This motivated the use of stratified sampling in the experimental protocol.

\subsection{Correlation and Redundancy Analysis}

\begin{figure}[H]
\centering

\caption{Correlation Heatmap}
\label{fig:pearsonheatmap}

A Pearson correlation heatmap (Figure~\ref{fig:pearsonheatmap}) revealed a very strong correlation ($r = 0.97$) between sleep quality index and deep sleep quality index. Point plots and one-way ANOVA tests were used to assess differences across diagnosis groups; neither variable showed statistically significant differences. However, \textit{sleep\_quality\_index} displayed slightly clearer group separation in the point plots and was therefore retained, while \textit{deep\_sleep\_quality\_index} was removed. The remaining numerical variables showed only weak to moderate correlations.


\subsection{Categorical Association Analysis}
Cramer's V was computed to assess associations between categorical predictors and diagnosis. Most predictors showed weak associations $V < 0.10$. However, pem\_present showed a perfect association $V = 1.000$, indicating it deterministically separates the diagnosis categories and therefore represents target leakage. It was removed prior to model training. After removal, all remaining categorical predictors showed negligible inter-associations (Figure~\ref{fig:cramers}).

\begin{figure}[H]

The results indicate weak associations between most categorical predictors and the diagnosis variable. For example, \textit{gender} and \textit{meditation\_or\_mindfulness} show almost no relationship with diagnosis, while \textit{work\_status}, \textit{exercise\_frequency}, and \textit{social\_activity\_level} exhibit only weak associations.

After removing this variable, Cramér's V values among the remaining categorical predictors were negligible ($V < 0.10$), indicating no strong associations between retained features.

\subsection{Experimental Protocol}

The dataset was split 80/20 with stratification. Model selection and hyperparameter tuning used stratified 5-fold CV, with macro F1-score as the primary metric. Categorical variables were encoded (nominal: one-hot; ordinal: ordinal; binary: 0/1), and numerical features were standardised for models sensitive to feature scale (Logistic Regression, k-NN). Tree-based models used raw values. All encoding and scaling were applied within a preprocessing pipeline during cross-validation to prevent data leakage.

\subsection{Selected ML Methods}

Three classifiers were evaluated: Logistic Regression (linear baseline), k-Nearest Neighbours (distance-based, non-parametric), and Random Forest (nonlinear ensemble). This selection allows comparison of linear, local-similarity, and ensemble-based approaches to understand which class of decision boundary best captures the diagnostic structure.

\subsection{Hyperparameter Search}

Hyperparameter tuning was conducted using GridSearchCV with stratified 5-fold cross-validation, optimising the Macro F1-score. Stratification preserves class distribution across folds, which is important due to moderate class imbalance. The search space included 
\textit{C}
 for Logistic Regression, 
\textit{k}
 for k-Nearest Neighbours, and \textit{the number of trees} and \textit{maximum depth} for Random Forest. The best configuration for each model was selected based on the highest cross-validated Macro F1-score

\section{Results}
\subsection{Decisions Based on Data Analysis}

Decisions based on the exploratory, correlation, and categorical association analyses presented in Sections~1.2--1.4 guided data preprocessing and feature selection.
Logistic Regression achieved a Macro F1-score of 0.774 with $C = 10$, while KNN performed worse (Macro F1 = 0.548, $k = 11$). Random Forest achieved the best performance (Macro F1 = 0.947 with {\textit{n\_estimators} = 500 and \textit{max\_depth} = 10), indicating that nonlinear relationships and feature interactions play an important role in predicting diagnosis.

\subsection{Model Comparison}

\begin{figure}[H]
\centering

\caption{Cross-validated model comparison}
\label{fig:modelcomparision}
\end{figure}


Logistic Regression achieved a Macro F1-score of 0.774 with $C=10$, indicating that weaker regularisation improved performance. 
The KNN model achieved a lower score of 0.548 with $k=11$, suggesting that local neighbourhood similarity alone was insufficient to separate the classes effectively. (Figure~\ref{fig:hyperparameter}).

Additionally, the feature space in this dataset is heterogeneous, containing a mix of numerical and encoded categorical variables with different scales and distributions, which further distorts distance calculations. The relatively large optimal k = 11 also suggests that the decision boundaries are not well-captured by local neighbourhood structure alone.

Random Forest achieved the highest cross-validated Macro F1-score of 0.947 with $n\_estimators=500$ and $max\_depth=10$. This indicates that nonlinear relationships and feature interactions play an important role in predicting diagnosis.

The large gap of Macro F1-scores confirms that nonlinear relationships and feature interactions drive diagnostic prediction. Logistic Regression's moderate score indicates some linear structure exists, while KNN's weak performance reflects the curse of dimensionality, in heterogeneous, high-dimensional feature spaces, distance-based reasoning becomes unreliable. Random Forest was selected as the final model.

\section{Final Model Evaluation}


The best-performing Random Forest model (\textit{n\_estimators} = 500, \textit{max\_depth = 10}) was evaluated on the held-out 20\%  test set, achieving a test accuracy of 0.963 and a Macro F1-score of 0.954. The CI class was predicted perfectly (F1 = 1.000), Depression achieved F1 = 0.955 with high recall (0.981), and the Both class showed slightly lower recall (0.857), with four observations misclassified as Depression, reflecting clinical overlap between these two categories. The close agreement between the cross-validated score (0.947) and the test score (0.954) indicates the model generalises well without substantial overfitting.

\subsection*{Conclusion}

This study evaluated machine learning methods for predicting diagnostic outcomes using demographic and clinical variables. Exploratory data analysis and feature screening were used to remove redundant and leakage-prone variables before modelling.

Among the evaluated methods, Random Forest achieved the best performance, indicating that nonlinear relationships and feature interactions are important for this task. The final model demonstrated strong performance on the held-out test set, suggesting good generalisation.

Future work could explore additional ensemble methods, alternative feature engineering strategies, or larger datasets to further improve predictive performance.
